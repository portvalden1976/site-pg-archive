# site-pg-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能来源于不同的项目、工具或临时需求，统一存放于此以便于访问和版本管理。

## 项目简介

`site-pg-archive` 是一个静态页面归档仓库。它不针对任何特定的网站或域名，而是作为一个通用的 HTML 页面集合，用于展示、测试或分享各种独立的前端内容。每个页面通常只依赖于自身的 HTML、CSS 和 JavaScript 资源，不依赖外部服务或数据库。

## 目录结构

```
site-pg-archive/
├── pages/          # 存放所有归档的 HTML 页面文件
├── assets/         # 页面共用的静态资源（如样式、脚本、图片等）
├── index.html      # 仓库主页，可用于列出或导航到各个归档页面
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件代表一个独立的归档页面。命名应简洁明了，反映页面内容或用途。
- **assets/**：存放多个页面可能共用的资源文件，避免重复。
- **index.html**：可选的入口页面，用于展示归档列表或提供导航。

## 页面归档说明

- 所有归档页面均为纯静态 HTML，可直接在浏览器中打开运行。
- 每个页面应尽量自包含，若需引用外部资源（如图片、字体），建议优先使用相对路径指向 `assets/` 目录。
- 归档页面不包含任何跟踪、分析或第三方服务调用，除非在页面本身中明确声明。
- 页面内容应合法、合规，不包含恶意代码、隐私收集或诱导性内容。

## 维护说明

- 添加新页面：在 `pages/` 目录下创建新的 HTML 文件或子目录，并确保其可正常访问。
- 更新页面：直接修改对应文件，并提交更新。建议在提交信息中简要说明变更内容。
- 删除页面：移除对应文件或目录，并更新 `index.html`（如有）。
- 资源管理：若新增页面使用了新的共用资源，请将其放入 `assets/` 目录中，并保持目录结构清晰。
